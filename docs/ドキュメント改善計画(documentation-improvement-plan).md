## 2. 改善方針

### 2.1 基本原則

1. **段階的改善**: すぐに必要なものから優先的に
2. **品質重視**: 新規作成時は「テスト・検証済み」のみ
3. **保守性重視**: ドキュメント間の冗長性を排除
4. **自動化**: パス確認等は自動スクリプト化
5. **継続性**: 更新ポリシーで長期的な品質保持

### 2.2 ドキュメント設計方針

```
【ドキュメント階層構造】

Level 1: 入口（初心者向け）
  └─ README.md
  └─ docs/quickstart/はじめに.md
  └─ docs/セットアップガイド詳細版（新規）

Level 2: ガイド（やることが明確）
  └─ docs/開発環境セットアップ（新規）
  └─ docs/トラブルシューティング詳細（拡張）
  └─ docs/09_運用/運用マニュアル

Level 3: 仕様（詳細情報）
  └─ docs/03_API仕様書/*
  └─ docs/02_設計書/*
  └─ docs/04_データベース/*

Level 4: 参考（背景知識）
  └─ docs/14_ADR/*
  └─ docs/ドキュメント完全インデックス（新規）
  └─ CLAUDE.md
```

## 3. セットアップ関連

### 3.1 Python 3.11 インストール失敗
**症状**: Python 3.11 が見つからない / 古いバージョンが起動される
**原因**: 複数のPythonバージョンが混在 / $PATH設定
**解決策**:
```bash
python3.11 --version  # 確認
which python3.11
export PATH="/usr/bin/python3.11:$PATH"  # 優先順位調整
```

### 3.2 依存関係インストール失敗
**症状**: `pip install -r requirements.txt` で失敗
**原因**: Python版違い / ディスク不足 / ネットワーク
**解決策**:
```bash
pip install --upgrade pip setuptools wheel
pip install -r requirements.txt --no-cache-dir
```

... (各バージョン・環境別)

## 4. フロントエンド関連

### 4.1 Bun ビルド失敗
**症状**: `bun run build` が失敗する
**原因**: Node版 / メモリ不足 / キャッシュ
**解決策**:
```bash
bun cache clean
bun install
NODE_OPTIONS="--max-old-space-size=4096" bun run build
```

rm -f data/database.db-shm
