# プロジェクト概要 (Project Overview)

## 1. プロジェクト名

**Personal Recipe Intelligence (PRI)**

## 2. プロジェクトの目的

個人が様々なソース（Webサイト、画像、手書きメモ等）からレシピ情報を収集し、構造化データとして一元管理するためのシステムを構築する。

## 3. 背景

### 3.1 課題
- レシピ情報が様々なサイトや媒体に分散している
- 海外レシピは言語の壁がある
- 手書きのレシピや雑誌の切り抜きはデジタル化されていない
- 既存のレシピ管理サービスは機能が限定的

### 3.2 解決策
- 複数のソースからレシピを自動収集
- OCR技術で画像からテキスト抽出
- 自動翻訳で海外レシピを日本語化
- 構造化データで一元管理

## 4. プロジェクトスコープ

### 4.1 スコープ内
- Webレシピの自動スクレイピング
- 画像からのOCRテキスト抽出
- DeepL APIによる自動翻訳
- レシピデータの正規化・構造化
- SQLiteによるデータ保存
- Svelte WebUIによる閲覧・検索

### 4.2 スコープ外
- モバイルアプリ開発
- マルチユーザー対応
- クラウドデプロイ
- レシピのソーシャル共有機能

## 5. ターゲットユーザー

- 個人（開発者本人）
- 料理好きで複数のソースからレシピを収集したい人
- 海外レシピを日本語で管理したい人

## 6. 主要機能

| 機能 | 説明 |
|------|------|
| Webスクレイピング | URL入力でレシピを自動抽出 |
| OCR | 画像からレシピテキストを抽出 |
| 自動翻訳 | 海外レシピを日本語化 |
| データ正規化 | 材料名・分量の統一 |
| 検索・フィルタ | タグ・キーワードで検索 |
| WebUI | レシピの閲覧・管理 |
