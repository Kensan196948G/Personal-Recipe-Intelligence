# ステークホルダー (Stakeholders)

## 1. 概要

本ドキュメントは、Personal Recipe Intelligence (PRI) プロジェクトに関わるステークホルダーを特定し、その役割・関心事・コミュニケーション方法を定義する。

## 2. ステークホルダー一覧

### 2.1 プライマリステークホルダー

| 名称 | 役割 | 関心事 |
|------|------|--------|
| プロジェクトオーナー | プロジェクト全体の意思決定 | 機能実現、品質、スケジュール |
| 開発者 | 設計・実装・テスト | 技術的実現性、開発効率 |
| エンドユーザー | システム利用 | 使いやすさ、機能の充実 |

### 2.2 セカンダリステークホルダー

| 名称 | 役割 | 関心事 |
|------|------|--------|
| ClaudeCode | AI開発支援 | CLAUDE.mdルール遵守 |
| SubAgents | 自動開発タスク実行 | タスク完遂、MCP制御 |
| レシピサイト運営者 | コンテンツ提供元 | 利用規約遵守 |

## 4. RACI マトリクス

| タスク | オーナー | 開発者 | ClaudeCode | SubAgents |
|--------|----------|--------|------------|-----------|
| 要件定義 | A | R | C | I |
| 設計 | A | R | R | I |
| 実装 | I | A | R | R |
| テスト | I | A | R | R |
| レビュー | A | R | C | I |
| デプロイ | A | R | I | I |
| 運用 | A | R | I | I |

- R: Responsible（実行責任）
- A: Accountable（説明責任）
- C: Consulted（相談）
- I: Informed（報告）

## 5. コミュニケーション計画

### 5.1 開発中

| 対象 | 方法 | 頻度 | 内容 |
|------|------|------|------|
| 開発者↔ClaudeCode | プロンプト | 常時 | 実装指示・確認 |
| ClaudeCode↔SubAgents | Hooks | 自動 | タスク連携 |
| 開発者 | Git commit | 機能完了時 | 変更記録 |

### 5.2 報告

| 報告内容 | 頻度 | 形式 |
|----------|------|------|
| 進捗状況 | 随時 | TodoWrite |
| 問題・課題 | 発生時 | GitHub Issues |
| 完了報告 | 完了時 | CHANGELOG |
